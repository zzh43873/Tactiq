# Tactiq - 地缘政治推演系统

基于多Agent技术的战略推演平台，利用AI进行地缘政治事件的情报收集、实体识别和多轮博弈推演。

## 核心功能模块

### 1. 情报收集系统 (Intelligence)

**文件位置**: `app/services/intelligence/`

#### 1.1 智能查询扩展 (QueryExpander)
- **功能**: 将用户命题扩展为多个互补的搜索查询
- **流程**:
  1. 查询扩展（当前背景/历史类比/影响推演/相关方分析）
  2. 实体预识别（识别可能涉及的所有实体）
  3. 关系分析（分析实体间关系）

#### 1.2 AI爬虫框架 (AICrawler)
- **功能**: 多源并发信息收集
- **数据源**:
  - GDELT (全球事件数据库，免费)
  - NewsAPI (新闻聚合，需API Key)
- **处理流程**:
  1. 多源并发收集
  2. 内容去重
  3. AI智能内容提取
  4. 实体识别与关系分析
  5. 相关性评分与排序

#### 1.3 情报收集器 (IntelligenceCollector)
- **核心流程**:
  ```
  检查缓存 → 查询扩展 → 实体预识别 → 迭代递归收集 → 实体识别与关系分析 
  → 态势理解 → 报告生成 → 保存缓存
  ```
- **迭代收集策略**:
  - 第1轮: 基础扩展查询
  - 第2-3轮: 对新实体进行专项搜索
  - 动态发现新实体直到收敛

### 2. 推演系统 (Simulation)

**文件位置**: `app/services/simulation/`

#### 2.1 推演编排器 (SimulationOrchestrator)
- 薄封装层，委托给 GeopoliticalGraph
- 接受命题字符串，返回格式化结果

#### 2.2 地缘政治图 (GeopoliticalGraph)
- **核心**: LangGraph状态图编排
- **LLM支持**: OpenAI / DeepSeek / SiliconFlow 自动切换
- **四阶段流水线**:

```
┌─────────┐     ┌─────────────────────┐     ┌─────────────────────┐
│  START  │────►│  Phase 1: 实体识别   │────►│  Phase 2: 实体画像    │
└─────────┘     │  (Entity Identifier)│     │  (Entity Profiler)  │
                │  - ReAct循环         │     │  - ReAct循环        │
                │  - 工具调用          │     │   - 背景资料收集      │
                └─────────────────────┘     └─────────────────────┘
                                                       │
                                                       ▼
┌─────────┐     ┌─────────────────────┐     ┌─────────────────────┐
│   END   │◄────│  Phase 4: 综合报告   │◄────│  Phase 3: 博弈推演    │
└─────────┘     │   (Synthesizer)     │     │  (Game Coordinator) │
                │  - 最终报告生成       │     │  - 并行实体Agent      │
                │  - 风险分析          │     │   - 多轮博弈          │
                └─────────────────────┘     └─────────────────────┘
```

#### 2.3 实体Agent (Entity Agent)
**内部鹰鸽辩论机制**:
```
┌─────────────────────────────────────────────────────────────┐
│                    单个实体决策流程                            │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  ┌───────────────┐                                          │
│  │  Hawk Advisor │──► 主张激进/对抗策略                        │
│  │  鹰派顾问      │    - 捍卫核心利益                          │
│  └───────┬───────┘    - 展示决心                             │
│          │            - 接受短期风险换取长期收益                │
│          ▼                                                  │
│  ┌───────────────┐                                          │
│  │  Dove Advisor │──► 主张克制/外交策略                        │
│  │  鸽派顾问      │    - 看到鹰派论点                          │
│  └───────┬───────┘    - 提出温和替代方案                       │
│          │            - 考虑国内成本                          │
│          ▼                                                  │
│  ┌───────────────────┐                                      │
│  │  Decision Maker   │──► 最终决策                           │
│  │  最终决策者         │    - 权衡双方观点                      │
│  └───────────────────┘    - 考虑国内约束                      │
│                           - 输出JSON格式决策                  │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

**决策维度**:
- Military (军事)
- Diplomatic (外交)
- Economic (经济)
- Energy (能源)
- Public Opinion (舆论)

### 3. 数据模型

**文件位置**: `app/models/`

| 模型 | 功能 | 关键字段 |
|------|------|----------|
| Entity | 地缘政治实体 | name, type, attributes, relationships |
| Event | 推演事件 | description, status, metadata |
| Simulation | 推演记录 | config, results, status, timestamps |
| IntelligenceCache | 情报缓存 | entities, articles, relationships, results |

### 4. 缓存与任务管理

**Redis应用**:
- 任务状态存储（推演/情报收集）
- 进度实时推送（WebSocket）
- 情报结果缓存（7天TTL）

## API 接口

### 推演接口

```http
POST /api/v1/simulation/run
Content-Type: application/json

{
  "proposition": "特朗普宣布出动地面部队攻击伊朗",
  "config": {
    "max_rounds": 3,
    "max_entities": 12
  }
}
```

**响应**:
```json
{
  "id": "uuid",
  "status": "running",
  "config": {...},
  "created_at": "2026-03-10T10:00:00"
}
```

### 情报收集接口

```http
POST /api/v1/intelligence/collect
Content-Type: application/json

{
  "event_description": "美国出兵伊朗的推演",
  "time_range": "past_30_days",
  "sources": ["gdelt", "newsapi"]
}
```

### 状态查询接口

```http
GET /api/v1/simulation/{simulation_id}
GET /api/v1/intelligence/task/{task_id}
```

## 部署方式

### Docker Compose 部署

```bash
# 启动所有服务
docker-compose up -d

# 服务组成:
# - frontend: 前端界面 (port 3000)
# - backend: API服务 (port 8000)
# - celery_worker: 异步任务处理
# - celery_beat: 定时任务调度
# - postgres: 数据库 (port 5432)
# - redis: 缓存 (port 6379)
```

### 环境变量配置

```bash
# LLM API配置 (至少配置一个)
OPENAI_API_KEY=sk-...
DEEPSEEK_API_KEY=...
SILICONFLOW_API_KEY=...

# 数据库
DATABASE_URL=postgresql+asyncpg://user:password@localhost:5432/geopolitics

# Redis
REDIS_URL=redis://localhost:6379/0

# 应用
APP_ENV=development
DEBUG=true
LOG_LEVEL=INFO
```

## 启动流程

1. **应用启动** (`main.py`):
   - 配置日志
   - 初始化FastAPI应用
   - 注册路由和中间件
   - 初始化数据库连接

2. **推演流程**:
   ```
   接收命题 → 创建任务 → 后台执行 → 状态更新 → 结果返回
   ```

3. **情报收集流程**:
   ```
   接收描述 → 查询扩展 → 实体预识别 → 多源爬取 → 内容分析 
   → 实体识别 → 关系分析 → 态势理解 → 报告生成 → 缓存保存
   ```

## 特性亮点

1. **智能查询扩展**: LLM驱动的多维度查询生成
2. **迭代递归收集**: 动态发现新实体，持续扩展情报
3. **鹰鸽辩论机制**: 模拟实体内部决策过程
4. **多轮博弈推演**: 并行实体Agent，多轮交互
5. **增量缓存更新**: 智能缓存策略，减少重复收集
6. **多LLM支持**: 自动切换多个LLM提供商
